# Self-Evolution 实操示例

> 来自第 11 章。一次典型的进化循环。

假设你刚完成一个"修 bug + 加测试"的任务：

1. **总结经验：** 这次修的是 token 过期问题，根因是判断逻辑缺少刷新机制。以后遇到"已过期但错误消息不明确"的情况，先用 `grep` 搜过期相关代码再下结论。
2. **沉淀知识：** 在项目 CHANGELOG 记下："auth/login.ts:45 token 判断缺少刷新分支，已修复。同类问题检查清单：搜索 `expir*` 关键词找到所有过期判断逻辑。"
3. **更新技能：** 如果这是第 3 次遇到"未先读代码就直接改"的坑，更新反惰性（第 3 章）的实战要点，添加本次教训。
4. **记录失败：** 第一次尝试用 `sed` 替换时没加引号导致命令报错——教训：shell 字符串包含特殊字符时必须用单引号包裹。

> **不是每次任务都要做满 4 步。** 简单任务（改个文案）只需第 1 步。复杂任务（修线上 bug）四步做满，重点在第 4 步——避免下次再踩同一个坑。
