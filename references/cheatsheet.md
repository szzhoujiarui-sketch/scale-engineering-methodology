# SCALE 速查表（CheatSheet）

> 1 分钟读完核心规则。完整方法论见 `SKILL.md`。

## 反幻觉三铁律

1. **没读文件不说内容** — 没有 read_file 调用，不得描述文件
2. **没跑命令不说通过** — 没有实际执行，不得说"测试通过"
3. **不确定标 UNCERTAIN** — 不确定的事实必须标注 `[UNCERTAIN]` + 说明缺失什么

## 工作流选择

| 你在做什么 | 工作流 | 耗时 |
|-----------|--------|------|
| 改文案错别字、调配置 | Express: fix → verify | 1-2min |
| 修 bug、加小功能 | Standard: plan → execute → verify | 5-15min |
| 新功能、大重构 | Full: explore → plan → execute → verify → consolidate | 20-60min |
| 线上事故、安全补丁 | Critical: 全门控 G1-G7 + 人工审查 | 30min+ |

## Honest Delivery（交付格式）

**Express（一行）：**
```
> ✅ [做了什么] | [验证方式] | [SCALE CHECK] G4,G5
```

**Standard 及以上（三栏）：**
```
✅ 完成内容: 具体改了什么（文件:行号）
🔄 验证结果: 门控通过情况（exit code / 来源可追溯）
❌ 未验证项: 什么没做、什么不确定
```

**铁律：** 未验证项 ≥ 3 个 → 停下补验证再交付（刚性）。

## 检测器优先级（9 选 3 必检）

| 优先级 | 检测器 | 一句话 | 何时触发 |
|--------|--------|--------|---------|
| 🔴 必检 | #6 Hallucination | 我编造了吗？ | 每次交付 |
| 🔴 必检 | #4 PrematureDone | 验证条件全过了吗？ | 每次交付 |
| 🔴 必检 | #9 SilentFail | 失败我报告了吗？ | 每次交付 |
| 🟡 高危 | #1 BruteRetry | 重试时分析原因了吗？ | ≥2 次重试 |
| 🟡 高危 | #7 ScopeCreep | 每行改动能追溯到需求吗？ | 多文件改动 |
| 🟡 高危 | #2 IdleTool | "我看看"真调了工具吗？ | 口头承诺无动作 |
| 🟢 可选 | #3 BusyLoop | 每步有新产出吗？ | 同类命令 ≥3 次 |
| 🟢 可选 | #5 BlameShift | 提供了方案还是只甩锅？ | 失败时 |
| 🟢 可选 | #8 CircularEdit | 这次改比上次有进展吗？ | 同文件 ≥3 次修改 |

## 自查决策树（5 步极简版）

```
第 1 步 诚实：我读过的真读了？我说的通过真跑了？ → 有一条否就停
第 2 步 范围：每处修改都能追溯到用户请求？顺手改的还原了吗？
第 3 步 进度：先扫 🔴必检 3 项（Hallucination / PrematureDone / SilentFail）
第 4 步 验证：选中的门控都实际执行了？exit code = 0？
第 5 步 交付：Honest Delivery 三栏准备好了吗？
→ 全部 ✅ → 可以交付
```
