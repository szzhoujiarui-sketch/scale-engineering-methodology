# 场景速查表（Scene Quick Reference）

> 按任务类型快速选择：该走哪个工作流 + 该过哪些门控。

---

## 代码场景

| 场景 | 工作流 | 必过门控 | 说明 |
|------|--------|---------|------|
| 改配置/拼写修正 | Express | G4, G5 | 1 文件，验证明确 |
| 修已知 bug | Standard (Bugfix) | G1, G4, G5 | 先复现再修复 |
| 加简单功能 | Standard | G2, G4, G5, G6 | plan → execute → verify |
| 新模块开发 | Full | G1-G6 | explore → plan → execute → verify → consolidate |
| 大规模重构 | Full | G1-G6 | 影响面分析必须先行 |
| 线上事故修复 | Critical | G1-G7 | 需人审，安全检查必过 |
| 安全补丁 | Critical | G1-G7 | 安全检查 + 回滚方案 |
| 生产数据库操作 | Critical | G1-G7 | 备份 + 回滚 + 人审 |

## 门控速查（非代码场景详细版）

### G4 — 事实核查 + 逻辑一致性

| 场景 | 具体检查 |
|------|---------|
| 客户邮件 | ① 订单号/产品规格是否查证过 ② 折扣率是否与实际政策一致 ③ 承诺的交付时间是否有依据 |
| 电商描述 | ① 产品参数与官网一致，不自创 ② 价格/促销信息与后台数据一致 ③ 用户评价引用真实存在 |
| 数据报告 | ① 每个结论对应一行数据 ② 同一指标在不同段落数值一致 ③ 图表标题与正文引用对应 |
| 调研报告 | ① 每个断言至少 1 个可追溯来源 ② A/B 方案优缺点自带引用 ③ 不确定信息标 [UNCERTAIN] |
| 财务分析 | ① 每个数字可追溯到原始数据 ② 计算公式正确（手动验算） ③ 汇率/口径统一 |

### G5 — 用户预览 + 交叉验证

| 场景 | 具体检查 |
|------|---------|
| 客户邮件 | ① 发送前让主人确认语气/风格 ② 关键数据交叉验算 |
| 电商描述 | ① 输出预览给用户确认卖点提炼是否准确 ② 与竞品对比数据交叉验证 |
| 数据报告 | ① 核心数据点手动验算 ② 趋势结论和数据匹配 |
| 调研报告 | ① 关键结论让用户确认理解正确 ② 多来源交叉验证同一事实 |
| 财务分析 | ① 关键比率/增长率让用户确认 ② 用另一种方法验算核心数字 |

### G6 — 结构完整性

| 场景 | 具体检查 |
|------|---------|
| 客户邮件 | ① 称呼/正文/结尾/签名完整 ② 附件/链接可访问 |
| 电商描述 | ① 标题/卖点/参数/售后四段完整 ② CTA 存在且位置正确 |
| 数据报告 | ① 标题/摘要/正文/结论四部分 ② 图表有标题，图例对应 |
| 调研报告 | ① 范围/方法/发现/结论四段不缺 ② 参考文献与引用一一对应 |
| 财务分析 | ① 摘要/数据/分析/建议/附录完整 ② 所有表格有表头和单位 |

---

## 使用方式

Agent 加载 SCALE 后，根据当前任务类型查此表：
1. 确定工作流层级（Express / Standard / Full / Critical）
2. 确定必过门控（G4 / G5 / G6 等）
3. 按门控速查的具体检查项逐项执行
