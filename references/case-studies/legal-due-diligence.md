## 1. 场景背景

### 1.1 任务描述

某私募股权投资基金委托 Agent 对一家拟收购标的公司 **"星辰科技股份有限公司"**（以下简称"目标公司"）进行法律尽职调查。尽调范围包括：

- 股权结构与历史沿革
- 重大诉讼与仲裁
- 行政处罚与合规记录
- 知识产权权属
- 重大合同审查

### 1.2 输入材料清单

Agent 通过读取以下材料开展工作（已通过 `read_file` 获取，非幻觉）：

| 材料编号 | 文件名称 | 来源 | 可信度 |
|---------|---------|------|-------|
| M1 | 目标公司最新营业执照 | 工商系统截图 | ✅ 官方 |
| M2 | 公司章程（2023年修订版） | 公司提供 | ✅ 内部 |
| M3 | 股东名册及出资证明 | 工商档案 | ✅ 官方 |
| M4 | 正在进行的诉讼清单（法务部提供） | 内部邮件 | ⚠️ 需交叉验证 |
| M5 | 裁判文书网检索报告 | 公开数据库 | ✅ 可追溯 |
| M6 | 行政处罚决定书（市监局 2024-017号） | 政府信息公开 | ✅ 官方 |
| M7 | 专利/商标注册证书清单 | 国家知识产权局 | ✅ 官方 |
| M8 | 前十大客户/供应商合同摘要 | 公司提供 | ⚠️ 需交叉验证 |

### 1.3 场景风险等级

```
风险评估:
  信息不对称 → 高（卖方提供材料可能不完整/美化）
  法律后果   → 高（遗漏重大诉讼可能导致投资决策错误）
  时间压力   → 中（尽调周期 2 周）
  推荐工作流 → Full（Full 工作流: G1 → G2 → G3 → G4 → G5 → G6 → G9）
```

### 2.3 G4 — 事实核查执行（核心演示）

> G4 非代码标准三要素：
> 1. **来源可追溯** — 每个事实必须标注具体来源
> 2. **内部逻辑无矛盾** — 不同来源的相同事实必须一致
> 3. **格式合规** — 排版、编号、引用规范

#### 2.3.1 核查 🔍1：股权结构矛盾

**原始矛盾**：

| 维度 | 股东名册 (M3) | 营业执照 (M1) |
|------|-------------|--------------|
| A股东持股 | 51%（2550 万） | — |
| B股东持股 | 29%（1450 万） | — |
| C股东持股 | 20%（1000 万） | — |
| 注册资本实缴状态 | 已全部实缴 | 显示"未实缴" |

**核查过程**：

```text
# 步骤 1: 从 M3 提取关键行
股东名册第 3 行: "A 股东出资 2550 万元，占比 51%，出资方式货币，已实缴"
股东名册第 7 行: "B 股东出资 1450 万元，占比 29%，出资方式货币，已实缴"
股东名册第 11 行: "C 股东出资 1000 万元，占比 20%，出资方式知识产权，已实缴"

# 步骤 2: 计算加总
2550 + 1450 + 1000 = 5000 → 与注册资本一致 ✅

# 步骤 3: 检查知识产权出资
知识产权出资 1000 万 → 需要评估报告 + 过户证明
M3 中未附评估报告 → 标记为 [UNCERTAIN]
C 股东是否已完成知识产权过户？→ M3 无记录

# 步骤 4: 交叉验证工商系统数据
[终端查询] 国家企业信用信息公示系统:
  目标公司: 星辰科技股份有限公司
  注册资本: 5000 万元   → 一致 ✅
  实缴状态: "未公示"    → 营业执照上"未实缴"可能是信息未更新
  股东信息: 已公示 A/B/C 三人 → 与 M3 一致 ✅
```

**事实核查结论**：

```
✅ 来源可追溯:
  - 股东比例: M3 第 3/7/11 行 + 工商公示系统查询结果
  - 实缴状态: 工商系统"未公示" ≠ "未实缴" → 已澄清

❌ 逻辑矛盾待解决:
  - C 股东知识产权出资 1000 万，M3 无评估报告 → 标记 [UNCERTAIN]
  - 需目标公司补充资产评估报告和过户证明

📌 最终结论:
  股权结构基本可信 ✅
  实缴状态非矛盾，是工商系统信息滞后 ✅
  C 股东出资证据不完整 ⚠️ → 建议尽调报告中列为"待补充项"
```

#### 2.3.2 核查 🔍2：诉讼差异

**原始矛盾**：

| 维度 | 法务部清单 (M4) | 裁判文书网 (M5) |
|------|----------------|----------------|
| 案件总数 | 5 | 3 |
| 作为被告案件 | 3 | 2 |
| 作为原告案件 | 2 | 1 |
| 案件标的总额 | 约 1200 万 | 约 800 万 |

**核查过程**：

```text
# 步骤 1: 逐一比对案件列表

M4（法务部）案件清单:
  [1] (2024)京0105民初12345号 — 买卖合同纠纷 — 被告 — 标的 500 万
  [2] (2024)沪0115民初67890号 — 软件服务合同纠纷 — 被告 — 标的 300 万
  [3] (2024)粤0304民初11111号 — 劳动争议 — 被告 — 标的 50 万
  [4] (2024)浙0106民初22222号 — 追偿权纠纷 — 原告 — 标的 200 万
  [5] (2024)苏0591民初33333号 — 专利侵权纠纷 — 原告 — 标的 150 万  ← NEW

M5（裁判文书网检索结果）:
  [A] (2024)京0105民初12345号 — 合同纠纷 — 已开庭待判决
  [B] (2024)沪0115民初67890号 — 合同纠纷 — 已达成调解
  [C] (2024)粤0304民初11111号 — 劳动争议 — 已判决（公司胜诉）

# 步骤 2: 差异识别
差异 A: M4 案件 [4] 追偿权纠纷（200 万）— 裁判文书网未收录
  原因分析: 可能刚立案，裁判文书网同步有延迟
  验证: 查询中国审判流程信息公开网 → 发现已立案但未公开文书 ✅

差异 B: M4 案件 [5] 专利侵权纠纷（150 万）— 裁判文书网未收录
  原因分析: 专利侵权案件可能涉及商业秘密，不公开审理
  验证: 查询中国审判流程信息公开网 → 发现"不公开审理"标记 ✅

差异 C: M4 案件 [3] 劳动争议标的 50 万 vs 判决结果公司胜诉
  一致性检查: 案件 [3] 员工索赔金额 50 万，M5 判决公司胜诉 → 无需支付
  → 逻辑一致 ✅
```

**事实核查结论**：

```
✅ 来源可追溯:
  - 每个案件的案号、法院、当事人、标的额都标注了来源（M4 条号 / M5 条号）
  - 差异案件通过第三方数据库（审判流程信息网）验证

✅ 逻辑一致:
  - M4 多出的 2 个案件有合理原因（延迟公开、不公开审理）
  - 劳动争议判决结果与索赔金额逻辑一致（胜诉则无需赔偿）

📌 最终结论:
  法务部清单可信度高 ✅
  标的总额差异（1200 万 vs 800 万）已解释：2 个未公开案件合计 350 万 ✅
  但专利侵权案件需关注 → 如果败诉可能影响核心技术 ⚠️
```

```text

  行业平均水平: 前五大客户占营收 40% → 65% 高于行业平均 → 集中度风险 ⚠️

# 步骤 3: 补充查询
目标公司是否有替代客户？
  → M8 中无相关信息
  → 标记为 [UNCERTAIN] 需公司补充

行业平均续约率:
  (未外部查询) 标记为 [UNCERTAIN] 避免编造数据
```

**事实核查结论**：

```
✅ 来源可追溯:
  - 合同信息: M8 摘要引用（标注行号）
  - 营收占比: 公司提供的财务报表

✅ 逻辑一致:
  - 剩余 42 天 > 30 天通知期 → 逻辑上仍在窗口期 ✅
  - 25% 营收占比 → 如果流失确实构成重大影响 ✅

❌ 未验证信息（标注 [UNCERTAIN]）:
  - 替代客户情况: M8 未提供
  - 行业平均续约率: 未外部查询，不编造
  - A 客户续约意向: 需通过管理层访谈确认

📌 结论:
  这是一个真实的合同到期风险，但确认程度有限 ⚠️
  尽调报告中应列为"需要管理层澄清项"
```

## 附录 A：G4 事实核查检查清单（通用版）

在**任何**非代码任务中执行 G4 时，逐项检查：

### A.1 来源可追溯

```
[ ] 每个数据点/引用都标记了来源编号（M1-M8 或类似）
[ ] 来源编号对应到具体行号或段落（而非"据文件显示"）
[ ] 使用了多个独立来源交叉验证（至少 2 个）
[ ] 单一来源的数据标注了可信度（官方/内部/待验证）
[ ] [UNCERTAIN] 标记用于：缺乏第三方验证、逻辑推测、缺失信息
```

### A.2 内部逻辑无矛盾

```
[ ] 同一事实在不同来源中比对过（如工商 vs 公司文件）
[ ] 发现的矛盾已分析原因（不仅是"不一致"）
[ ] 矛盾已解决或标记为未解决
[ ] 数值计算加总验证过（如股权比例=100%、诉讼标的=各案之和）
[ ] 因果推导合理（如"判决胜诉"→"无需赔偿"）
```

### A.3 格式合规

```
[ ] 引用格式统一（来源+行号）
[ ] 表格/列表排版清晰
[ ] 核查结论标准化（来源可追溯 / 逻辑一致 / 最终结论）
[ ] 无 AI 生成的模板占位符
[ ] [OWNER] 标记用于超出范围但相关的发现
```
