# 案例：电商运营场景 — Standard 工作流

> **SCALE 工程方法论 v10.3 · 实战案例**
> 场景类型：Standard 工作流（非代码任务）
> 对应章节：第 7 章（Standard 层级）
> 关联门控：G4（事实核查）, G5（用户预览）, G6（结构完整性）

---

## 1. 场景速览

| 项目 | 内容 |
|------|------|
| 用户原始输入 | "帮我写一封 618 大促的客户召回邮件" |
| 任务类型 | 营销文案撰写 |
| 风险等级 | 中（涉及客户沟通、促销信息、品牌形象） |
| 推荐工作流 | **Standard：plan → execute → verify** |
| 耗时参考 | 5-15 分钟 |

---

## 2. 使用前提检查（是否适用 Standard？）

| 前提条件 | 检查结果 |
|---------|---------|
| 涉及 3 个以上文件？ | ❌ 否（单封邮件） |
| 需要探索或设计决策？ | ⚠️ 是（需要了解促销政策和客户分群） |
| 验证方式明确？ | ✅ 是（用户预览 + 数据核查） |

**结论：需要 plan 阶段了解促销信息，走 Standard 工作流。**

---

## 3. Standard 工作流执行过程

### 3.1 plan 阶段

```
步骤 1: 确认促销信息
验证: 用户确认折扣率、满减规则、活动时间

步骤 2: 确认客户分群
验证: 用户确认召回对象（30天未下单客户）

步骤 3: 撰写邮件初稿
验证: 用户预览确认语气和内容

步骤 4: 最终检查
验证: G4 事实核查 + G5 用户确认 + G6 结构完整
```

### 3.2 execute 阶段

```
Agent: 我需要确认几个关键信息才能写出准确的邮件：

1. 618 的具体促销政策是什么？
   - 折扣率：全场 8 折？还是部分商品？
   - 满减规则：满 300 减 50？
   - 活动时间：6月1日-6月18日？

2. 召回对象是哪些客户？
   - 30 天未下单？60 天？
   - 是否区分 VIP 和普通客户？

3. 邮件风格偏好？
   - 正式商务风？
   - 轻松亲切风？

用户: 全场 8 折，满 300 减 50，6月1日-18日。30天未下单客户，不分 VIP。
      风格轻松一点，像朋友推荐。

Agent: 明白。开始撰写邮件初稿...

[撰写邮件内容]

Agent: 邮件初稿如下，请确认：

---

主题：好久不见，618 给你留了个专属折扣 🎁

Hi [客户名]，

好久没见你来了，怪想你的。

618 来了，给你留了个小福利：
- 全场 8 折
- 满 300 再减 50
- 活动时间：6月1日 - 6月18日

上次你看的 [热销商品] 还在，趁活动入手刚好。

[立即选购] 按钮

有任何问题随时找我～

[品牌名] 团队

---
```

### 3.3 verify 阶段（门控检查）

```
G4 — 事实核查：
✅ 折扣率 "全场 8 折" — 与用户确认的促销政策一致
✅ 满减规则 "满 300 减 50" — 与用户确认一致
✅ 活动时间 "6月1日-18日" — 与用户确认一致
✅ 无自创数据（没有编造转化率、用户数等）

G5 — 用户预览：
✅ 用户确认语气轻松亲切，符合要求
✅ 用户确认内容准确

G6 — 结构完整性：
✅ 主题行存在
✅ 正文有称呼、内容、CTA、签名
✅ CTA 按钮位置正确
✅ 无留空字段

✅ 完成内容:
  - 618 客户召回邮件初稿已完成
  - 促销信息与用户确认一致

🔄 验证结果:
  - G4 事实核查: pass（所有数据与用户确认一致）
  - G5 用户预览: pass（用户已确认）
  - G6 结构完整: pass（邮件格式完整）

❌ 未验证项:
  - 无
```

---

## 4. 关键教训

1. **先确认再写** — 促销信息、客户分群、风格偏好必须在 plan 阶段确认
2. **事实核查是核心** — 邮件中的折扣率、满减规则、活动时间必须与实际政策一致
3. **用户预览不可省** — 语气和风格是主观判断，必须用户确认
4. **不要自创数据** — "转化率提升 30%"这种没有依据的话不能写
